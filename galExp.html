During the doctoral program

Ballbot


This ballbot is built at IRMS lab, Kyung Hee University (2017-2018). In this system, mechanical parts are designed by the FUSIONENC company. Then, my role is to set it up from scratch and propose robust control algorithms (TSMC, MSSP, and IJCAS) to govern the balance and transfer of ballbot. Then, these algorithms are simulated in Matlab and are converted into C++.

  • Finally, a control software (MFC C++) is programmed using the multithreading technique to fuse the sensor data and generate control signals to motors. The connection diagram, the control architecture diagram can be observed at TSMC, MSSP, and IJCAS papers.

During undergraduate program

Build a remote control framework for a mobile manipulator platform. (Funded by HCMUT).


This robot includes a mobile platform attached with a three DOFs robot arm. The system comprises a joystick, embedded boars, IMU, two cameras, wifi router, power boards, voltage level shifting, and mixing signal boards. Most ICs and MCUs are supported by Texas Instruments. The robot can be remotely controlled using wifi. The main control program is written by C++. It is running on an embedded board (BeagleBone Black-Linux). The operator can control the system with a joystick. My main tasks are as follows.

  • Design printed circuit boards for the power stage (high current), voltage level shifting, mixing control signals, and self-safety.

  • Write a socket server on the BeagleBone Black (Python) to receive and handle the control signals from the operator. This socket will activate the received commands and get the data from other connected sensors using the bash script and low-level assembly languages (to inherit the real-time action of the PRU module).

  • Write a socket client on the remote computer (C#) to decode the signal from the joystick and send it wirelessly to the server. In addition, the data from cameras is also collected.

  • Forward/inverse kinematics is used to control a 3DOF arm is proposed. Also, the OpenGL library is used to simulate the robot arm motion.

  • The OpenCV library is invoked to do object detection for the robot arm.
